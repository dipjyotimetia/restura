**Requirements:** Node.js **24 or later**. The package pins `npm@11.17.0` as its package manager.

```bash
npm install
```

## Core commands

| Command | What it does |
| --- | --- |
| `npm run dev` | Starts the Vite dev server |
| `npm run build` | Production build |
| `npm run test` | Vitest, interactive |
| `npm run test:run` | Vitest, single run |
| `npm run lint` | Biome lint across `src`, `shared`, `electron/main`, `worker`, `echo`, `echo-local`, `cli`, `extension/chrome`, `extension/vscode`, `tests`, `scripts` |
| `npm run format` | Biome format, write mode |
| `npm run validate` | Coverage-aware local shipping gate: `type-check:all` + Biome + generated-artifact checks + `test:ci` + CLI tests |

## The type-check vs type-check:all gotcha

`npm run type-check` runs `tsc --noEmit` against the **root** `tsconfig.json` only. That config's `exclude` array is:

```json
["node_modules", "dist", "out", "worker", "electron/main", "cli", "extension", "docs-site"]
```

<Aside type="caution">
A green `npm run type-check` tells you nothing about whether `worker/`, `electron/main/`, `cli/`, `extension/`, or `docs-site/` actually compile — they're all excluded from that command's scope.
</Aside>

`npm run type-check:all` is the command that actually covers them: it runs the base `type-check`, then chains five more `tsc --noEmit -p <tsconfig>` project builds (`electron/tsconfig.json`, `src/features/http/tsconfig.json`, `worker/tsconfig.json`, `echo/tsconfig.json`, `echo-local/tsconfig.json`), plus the `cli`, `extension` (Chrome), and `restura-vscode` workspace type-checks. `npm run validate` — and CI — use `type-check:all`, not the plain `type-check`.

Local validation is necessary but not the complete shipping verdict. GitHub's
`merge-gate` additionally aggregates the shipped self-host image and API/SPA smoke,
browser and Electron E2E, both extension surfaces, documentation, and cross-OS
desktop packaging for the exact branch SHA.
