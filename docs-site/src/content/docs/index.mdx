## Why Restura

<CardGrid stagger>
  <Card title="Three platforms, one renderer" icon="laptop">
    Web app on Cloudflare Pages, native desktop apps for macOS / Windows / Linux, and a
    self-hostable Docker image. The same React UI ships everywhere.
  </Card>
  <Card title="Privacy first" icon="seti:lock">
    No account, no cloud sync, no telemetry. Collections, history and secrets live on your machine —
    IndexedDB on web, OS keychain on desktop.
  </Card>
  <Card title="Scriptable, sandboxed" icon="seti:code-search">
    Pre-request and test scripts run in an isolated QuickJS WASM sandbox using the native `rs.*` API
    — fully Postman `pm.*` compatible. No DOM, no network escape.
  </Card>
  <Card title="Chain requests with workflows" icon="seti:plan">
    Build request graphs with branching, switches, loops, and parallel steps. Extract variables with
    JSONPath, regex, or headers. Run anywhere — including in CI.
  </Card>
  <Card title="Built-in AI assistant" icon="seti:notebook">
    Chat with OpenAI, Anthropic, or OpenRouter. The assistant can read the current request and
    response for context — with redaction baked in.
  </Card>
  <Card title="Speaks MCP both ways" icon="puzzle">
    Restura proxies any MCP server, and it can *be* an MCP server — exposing collections,
    environments, and history to agents like Claude.
  </Card>
</CardGrid>
