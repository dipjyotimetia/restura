Socket.IO isn't plain WebSocket — it has its own framing, namespacing, ack callbacks, and the option to fall back to long-polling. Restura speaks the protocol natively.

## What works

- **Connect** to any Socket.IO server, with or without namespacing (`/admin`, etc.).
- **Auth payload + query params** on connect — editable while disconnected; the auth map is sent in the handshake `auth` field, query params on the connection URL.
- **Emit events** — pick an event name, send a JSON / string / number payload.
- **Listen** — subscribe to event names; live messages stream into the transcript.
- **Acks** — toggle **Ack** in the compose bar to request the callback-based response; the resolved ack args stream back into the transcript with an `ok` or `timeout` status (15s).
- **Reconnect** — on/off, with backoff settings.

## Transports

The web build connects **browser-direct** with `socket.io-client`, so emits, listeners, and ack callbacks all work without a proxy. The desktop app runs the same client in the Electron main process — reach for it when you need custom `extraHeaders` on the handshake, which browsers can't set.

## Tips

- Acks behave identically on web and desktop: `socket.emit('event', payload, (response) => ...)` resolves into the transcript.
- Use the **[desktop app](/overview/install/)** when your handshake needs `extraHeaders` the browser won't allow.
