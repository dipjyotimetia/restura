## What a contract test checks

`tests/contract/` verifies **cross-backend parity of the shared protocol core** ŌĆö not any one backend in isolation. Each test runs the exact same `RequestSpec` through `executeHttpProxy` (or `executeHttpProxyStreaming`) twice, once per `Fetcher` adapter, against one real local upstream, and asserts the two `ExecuteResult`s match.

Two fetchers stand in for the real backends:

- **`workerFetcher`** ŌĆö uses `globalThis.fetch`, mirroring `worker/handlers/proxy.ts`.
- **`electronFetcher`** ŌĆö uses undici, mirroring `electron/main/handlers/http-handler.ts`. It adapts undici's response shape to the shared `FetcherResponse` contract (headers collapsed to `Record<string, string | string[]>`, `statusText` hard-coded to `''` since undici doesn't expose it).

The upstream is a real `node:http` server (`tests/contract/upstream.ts`) with fixture routes ŌĆö headers echo, JSON roundtrip, redirect chains, chunked transfer-encoding, arbitrary status codes ŌĆö not a mock of the shared code and not the Cloudflare runtime.

`http-proxy.contract.test.ts` runs buffered fixtures (headers, query params, JSON POST, 404, 500, chunked body, redirect chains) through both fetchers and asserts pairwise equality. `http-proxy-streaming.contract.test.ts` does the same for the streaming path (`executeHttpProxyStreaming`) that SSE/NDJSON/chunked downloads use ŌĆö asserting identical bytes on both rails, identical rejection of a disallowed method, and identical handling of a non-2xx status. The buffered test never exercises the streaming code path, so the streaming file exists specifically to catch stream-framing/decoding parity breaks that would otherwise only surface in end-to-end tests.

Run it with:

```bash
npm run test:contract
```

## Codegen-drift gates

Two more gates catch a different class of bug: generated files that have gone stale relative to their source of truth.

- **`npm run verify:opencollection-types`** ŌĆö regenerates `src/lib/opencollection/spec-types.ts` from the OpenCollection JSON Schema, then fails if that regeneration produces a diff. If this fails, the schema changed but the generated types weren't regenerated and committed.
- **`npm run capabilities:check`** ŌĆö regenerates `docs/CAPABILITY_MATRIX.md` from `src/lib/shared/capabilities.ts` and fails on any diff, for the same reason.

Both are "generated file is out of date" checks, not functional tests ŌĆö fix them by running the corresponding `gen:*` script and committing the result, not by hand-editing the generated file.

## Local validation and the complete CI gate

`npm run validate` is the coverage-aware local slice of the pipeline. It chains
`type-check:all` ŌåÆ `lint` ŌåÆ `format:check` ŌåÆ
`verify:opencollection-types` ŌåÆ `capabilities:check` ŌåÆ `test:ci` ŌåÆ the CLI
workspace's own tests. The explicit command also records content-bound evidence
for the repository's Codex stop policy.

Run the whole thing locally before opening a PR:

```bash
npm run validate
```

`validate` is one slice of a larger pipeline. GitHub's fail-closed `merge-gate`
also requires the web/Worker build, shipped self-hosted image and API/SPA smoke,
bundle-size checks, Playwright end-to-end (web and Electron), Chrome and VS Code
extension validation, the docs build, and cross-OS Electron packaging. It runs
on pull requests and pushes to `main`, and releases accept only a successful
gate produced for the exact candidate SHA. See
[`docs/CI_CD.md`](https://github.com/dipjyotimetia/restura/blob/main/docs/CI_CD.md)
for the workflow and live-ruleset details.
